Cómo crear una máquina virtual

40 minutos 1 crédito
Calificar lab

GSP001

Descripción general

Google Compute Engine le permite crear máquinas virtuales que ejecutan diferentes sistemas operativos, incluidos varios tipos de Linux (Debian, Ubuntu, Suse, Red Hat, CoreOS) y Windows Server, en la infraestructura de Google. Puede ejecutar miles de CPU virtuales en un sistema que se diseñó para ser rápido y ofrecer una coherencia sólida de rendimiento.

En este lab práctico, aprenderá a crear instancias de máquinas virtuales de varios tipos de máquinas con Google Cloud Platform (GCP) Console y la línea de comandos de gcloud. También aprenderá a conectar un servidor web NGINX a su máquina virtual.

Aunque puede copiar comandos del lab y pegarlos en el lugar apropiado fácilmente, los estudiantes deberían escribir los comandos para reforzar su comprensión de los conceptos básicos.

Actividades

  • Crear una máquina virtual con GCP Console
  • Crear una máquina virtual con la línea de comandos de gcloud
  • Implementar un servidor web y conectarlo a una máquina virtual

Requisitos previos

  • Se recomienda estar familiarizado con editores de texto estándares de Linux, como vim, emacs o nano.

Información sobre las regiones y zonas

Algunos recursos de Compute Engine se encuentran en regiones o zonas. Una región es una ubicación geográfica específica donde puede ejecutar sus recursos. Cada región tiene una o más zonas. Por ejemplo, la región “us-central1” corresponde a una región en el centro de Estados Unidos que tiene las zonas us-central1-a, us-central1-b, us-central1-c y us-central1-f.

Los recursos ubicados en una zona se denominan recursos zonales. Las instancias de máquinas virtuales y los discos persistentes se encuentran en una zona. Un disco persistente y una instancia de máquina virtual deben ubicarse en la misma zona para poder vincularlos. De forma similar, para asignar una dirección IP estática a una instancia, ambas deben ubicarse en la misma región.

Consulte la siguiente documentación para saber más sobre el tema y ver una lista completa de regiones y zonas.

Cree una instancia nueva con gcloud

En lugar de utilizar GCP Console para crear una instancia de máquina virtual, puede usar la herramienta de línea de comandos gcloud, que viene preinstalada en Google Cloud Shell. Cloud Shell es una máquina virtual basada en Debian cargada con todas las herramientas de desarrollo que necesitará (gcloud, git y otras) y ofrece un directorio principal persistente de 5 GB.

Si en el futuro desea probar esto en su máquina, lea la guía de la herramienta de línea de comandos gcloud.

En Cloud Shell, cree una instancia de máquina virtual nueva desde la línea de comandos mediante gcloud; reemplace [YOUR_ZONE] por una de las opciones de zona que eligió antes:

gcloud compute instances create gcelab2 --machine-type n1-standard-2 --zone [your_zone]

(Resultado)

Created [...gcelab2].
NAME     ZONE           MACHINE_TYPE  ...    STATUS
gcelab2  us-central1-c  n1-standard-2 ...    RUNNING

Haga clic en Revisar mi progreso a continuación para verificar que está avanzando en este lab.

Crear una instancia nueva con gcloud

La instancia creada tiene estos valores predeterminados:

  • La imagen más reciente de Debian 9 (stretch)
  • El tipo de máquina n1-standard-2. En este lab, puede seleccionar uno de estos otros tipos de máquina si lo desea: n1-highmem-4 o n1-highcpu-4; cuando trabaje en un proyecto fuera de Qwiklabs, también puede especificar un tipo personalizado de máquina
  • Un disco raíz persistente con el mismo nombre que la instancia; el disco se adjunta automáticamente a la instancia

Ejecute gcloud compute instances create --help para ver todos los valores predeterminados.

Nota: Puede configurar la región y las zonas que gcloud utiliza de forma predeterminada si siempre trabaja dentro de una región o zona y no desea agregar la marca --zone cada vez. Para ello, ejecute los siguientes comandos:

gcloud config set compute/zone ...

gcloud config set compute/region ...

Para salir de help, presione Ctrl + c.

Consulte sus instancias. Seleccione el menú de navegación > Compute Engine > Instancias de VM. Debería ver las 2 instancias que creó en este lab.

Finalmente, también puede establecer una conexión SSH a su instancia mediante gcloud. Asegúrese de agregar su zona o de omitir la marca --zone si estableció la opción globalmente:

gcloud compute ssh gcelab2 --zone [YOUR_ZONE]
