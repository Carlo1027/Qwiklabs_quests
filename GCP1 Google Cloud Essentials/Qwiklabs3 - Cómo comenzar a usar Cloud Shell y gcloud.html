Cómo comenzar a usar Cloud Shell y gcloud

45 minutos 1 crédito
Calificar lab

GSP002

Descripción general

Google Cloud Shell le proporciona acceso de línea de comandos de gcloud a recursos de procesamiento alojados en Google Cloud Platform. Cloud Shell es una máquina virtual basada en Debian que cuenta con un directorio principal persistente de 5 GB, lo que facilita la administración de los proyectos y recursos de GCP. El gcloud del SDK de Cloud y otras herramientas que necesita vienen preinstalados en Cloud Shell, lo que le permite ponerse en marcha rápidamente.

En este lab práctico, aprenderá a conectarse a los recursos de procesamiento alojados en Google Cloud Platform a través de Cloud Shell, con la línea de comandos de gcloud.

Se incentiva a los estudiantes a que ellos mismos escriban los comandos, ya que eso refuerza los conceptos básicos. Muchos labs incluyen un bloque de código que contiene los comandos necesarios. Durante el lab, puede copiar y pegar fácilmente los comandos del bloque de código en los lugares apropiados.

Actividades

  • Practicar el uso de los comandos de gcloud
  • Conectarse a los servicios de procesamiento alojados en Google Cloud Platform

Requisitos previos

  • Se recomienda estar familiarizado con editores de texto estándares de Linux, como vim, emacs o nano.

Configuración

Antes de hacer clic en el botón Comenzar lab

Lea estas instrucciones. Los labs tienen un tiempo límite; no puede ponerlos en pausa. El cronómetro, que comienza a funcionar cuando hace clic en Comenzar lab, indica por cuánto tiempo estarán los recursos en la nube a su disposición.

Este lab práctico de Qwiklabs le permitirá llevar a cabo las actividades correspondientes en un entorno de nube real, no en uno de simulación o demostración. Para ello, recibe nuevas credenciales temporales que utilizará para acceder a Google Cloud Platform durante el transcurso del lab.

Lo que necesita

Para completar este lab, necesitará lo siguiente:

  • Acceso a un navegador de Internet estándar. Se recomienda el navegador Chrome
  • Tiempo para completar el lab

Nota: Si ya tiene un proyecto o una cuenta personal de GCP, no los use para este lab.

Cómo iniciar su lab y acceder a Console

  1. Haga clic en el botón Comenzar lab. Si debe pagar por el lab, se abrirá una ventana emergente para que seleccione su forma de pago. En la parte izquierda, verá un panel con las credenciales temporales que debe usar para este lab.

  2. Copie el nombre de usuario y, luego, haga clic en Abrir Google Console. El lab inicia los recursos y abre otra pestaña que muestra la página Seleccione una cuenta.

    Sugerencia: Abra las pestañas en ventanas separadas, una junto a la otra.

  3. En la página Seleccione una cuenta, haga clic en Usar otra cuenta.

  4. Se abrirá la página de acceso. Pegue el nombre de usuario que copió del panel Detalles de conexión. Luego, copie y pegue la contraseña.

    Importante: Debe usar las credenciales del panel Detalles de conexión. No use sus credenciales de Qwiklabs. Si tiene una cuenta propia de GCP, no la use para este lab (evite incurrir en cargos).

  5. Avance por las siguientes páginas haciendo clic en ellas:

    • Acepte los Términos y Condiciones.
    • No agregue opciones de recuperación o autenticación de dos factores (esta es una cuenta temporal).
    • No se registre para obtener pruebas gratuitas.

Después de un momento, se abrirá GCP Console en esta pestaña.

Nota: Para ver el menú con una lista de productos y servicios de GCP, haga clic en Menú de navegación, en la esquina superior izquierda, junto a "Google Cloud Platform".

Inicie Cloud Shell

Abra una sesión de Cloud Shell. Para ello, haga clic en el ícono ubicado en la esquina superior derecha de GCP Console:

Luego, haga clic en Iniciar Cloud Shell:

Después de activar Cloud Shell, puede usar la línea de comandos a fin de invocar el comando de gcloud del SDK de Cloud o cualquier otra herramienta disponible en la instancia de la máquina virtual. Más adelante en el lab, usará su directorio $HOME, que se utiliza en el almacenamiento en disco persistente, para almacenar archivos en los proyectos y hacer que persistan de una sesión de Cloud Shell a otra. Su directorio $HOME es privado, por lo que otros usuarios no pueden acceder a él.

Información sobre las regiones y zonas

Algunos recursos de Compute Engine se encuentran en regiones o zonas. Una región es una ubicación geográfica específica donde puede ejecutar sus recursos. Cada región tiene una o más zonas. Por ejemplo, la región “us-central1” corresponde a una región en el centro de Estados Unidos que tiene las zonas us-central1-a, us-central1-b, us-central1-c y us-central1-f.

Los recursos ubicados en una zona se denominan recursos zonales. Las instancias de máquinas virtuales y los discos persistentes se alojan en una zona. Un disco persistente y una instancia de máquina virtual deben ubicarse en la misma zona para poder vincularlos. De forma similar, para asignar una dirección IP estática a una instancia, ambas deben ubicarse en la misma región.

Para obtener más información sobre las regiones y zonas y ver una lista de todas ellas, consulte esta documentación.

Las regiones y zonas predeterminadas se establecen con los siguientes valores:

google-compute-default-zone google-compute-default-region

Para consultar la configuración de sus regiones y zonas predeterminadas, ejecute el siguiente comando de gcloud, en el que deberá reemplazar <your_project_id> por el ID de su proyecto, que puede ver en la página principal de Console o en la pestaña de Qwiklabs desde la que inició el lab:

gcloud compute project-info describe --project <your_project_ID>

Usará la zona (google-compute-default-zone) que obtenga del resultado más adelante en el lab.

Busque los valores de metadatos de la zona y región predeterminadas en la respuesta. Si la respuesta no contiene las claves y los valores google-compute-default-region y google-compute-default-zone, significa que no se estableció ninguna zona o región predeterminada.

Cómo inicializar el SDK de Cloud

La CLI gcloud forma parte del SDK de Google Cloud. A fin de utilizarla, deberá descargar y, luego, instalar el SDK en su propio sistema. A continuación, deberá ejecutar gcloud init para inicializarlo.

La CLI gcloud está disponible automáticamente en Cloud Shell. Dado que utilizará Cloud Shell en este lab, no necesita instalar gcloud de forma manual.

Configure variables de entorno

Las variables de entorno son variables que definen su entorno. Le permiten definir sus propias variables y ahorrar tiempo cuando escribe secuencias de comandos que contienen API o archivos ejecutables.

Cree algunas variables de entorno:

export PROJECT_ID=<your_project_ID>

Establezca su variable de entorno ZONE (utilice el valor para la zona que obtuvo mediante el comando anterior):

export ZONE=<your_zone>

Verifique que sus variables se hayan establecido correctamente:

echo $PROJECT_ID
echo $ZONE

Cree una máquina virtual con gcloud

Cree una nueva instancia de máquina virtual mediante gcloud. En el próximo comando, usará lo siguiente:

  • gcloud compute, que le permite administrar con facilidad sus recursos de Google Compute Engine en un formato más amigable que si utiliza la API de Compute Engine
  • instances create para crear una nueva instancia

Ejecute el siguiente comando a fin de crear su VM:

gcloud compute instances create gcelab2 --machine-type n1-standard-2 --zone $ZONE
  • El nombre de la VM es “gcelab2”.
  • Usará la marca --machine-type para especificar el tipo de máquina como “n1-standard-2”.
  • Usará la marca --zone para especificar que se cree en la zona que definió con su variable de entorno.

(Resultado)

Si omite la marca --zone, gcloud puede inferir la zona que desea según sus propiedades predeterminadas. Si en el comando create no especifica otra configuración necesaria sobre la instancia, como el tipo de máquina y la imagen, se establecerán los valores predeterminados.

Pruebe la tarea completada

Haga clic en Revisar mi progreso para verificar la tarea realizada. Si creó correctamente una máquina virtual con gcloud, verá una puntuación de evaluación.

Crear una máquina virtual con gcloud

Puede consultar los valores predeterminados si solicita que se muestre la ayuda del comando create, como se indica a continuación:

gcloud compute instances create --help

Cómo usar los comandos de gcloud

Para consultar los lineamientos de uso simples que ofrece gcloud, agregue la marca -h (que representa “help”; es decir, “ayuda”) al final de cualquier invocación de gcloud.

Ejecute el siguiente comando en Cloud Shell:

gcloud -h

Agregue la marca --help o ejecute el comando gcloud help a fin de obtener ayuda más detallada. En Cloud Shell, ejecute lo siguiente:

gcloud config --help

Presione la tecla Intro o la barra espaciadora para desplazarse por el contenido de ayuda.

Escriba q para salir del contenido.

Ahora, ejecute el siguiente comando:

gcloud help config

Puede ver que los comandos gcloud config --help y gcloud help config son equivalentes. Ambos proporcionan ayuda completa y detallada.

Las marcas globales de gcloud determinan el comportamiento de los comandos en un nivel por invocación. Las marcas anulan cualquier valor configurado en las propiedades de SDK.

Visualice la lista de configuraciones en su entorno:

gcloud config list

Llame al siguiente comando para ver cómo están configuradas otras propiedades:

gcloud config list --all

Obtenga una lista de sus componentes:

gcloud components list

Aquí verá qué componentes están listos para que los utilice en este lab. A continuación, instalará un nuevo componente.

Autocompletado

gcloud interactive incluye sugerencias automáticas para comandos y marcas, y muestra fragmentos de ayuda intercalados en la sección más baja a medida que se escribe el comando.

La información estática, como los nombres de comandos y subcomandos, y los nombres de marcas y valores de marcas enumerados, se autocompletan mediante el uso de menús desplegables.

Instale los componentes Beta:

gcloud components install beta

Ingrese al modo gcloud interactive:

gcloud beta interactive

Cuando utilice el modo interactivo, presione la tecla de tabulación para completar la ruta de archivo y los argumentos de recursos. Si aparece un menú desplegable, use la tecla de tabulación para desplazarse por la lista y la barra espaciadora para seleccionar su opción.

Pruébelo. Comience a escribir el siguiente comando y utilice el autocompletado para finalizarlo:

gcloud compute instances describe <your_vm>

En la parte inferior de Cloud Shell, puede ver la combinación de teclas para activar o desactivar esta función. Pruebe activarla o desactivarla con F2:

F2:help:STATE Activa o desactiva la sección de ayuda activa, ACTIVADA cuando está habilitada, DESACTIVADA cuando está inhabilitada.

Establezca una conexión SSH a su instancia de VM

gcloud compute facilita la conexión a las instancias. El comando gcloud compute ssh proporciona un wrapper para SSH, que se encarga de la autenticación y de mapear el nombre de la instancia a la dirección IP.

Utilice gcloud compute ssh para establecer una conexión SSH a su VM:

gcloud compute ssh gcelab2 --zone $ZONE

(Resultado)

WARNING: The public SSH key file for gcloud does not exist.
WARNING: The private SSH key file for gcloud does not exist.
WARNING: You do not have an SSH key for gcloud.
WARNING: [/usr/bin/ssh-keygen] will be executed to generate a key.
This tool needs to create the directory
[/home/gcpstaging306_student/.ssh] before being able to generate SSH Keys.

Escriba “Y” para continuar:

Do you want to continue? (Y/n)

Presione la tecla Intro para avanzar por la sección de la frase de contraseña y dejarla vacía.

Generating public/private rsa key pair.
Enter passphrase (empty for no passphrase)

Dado que aquí no necesita hacer nada, cancele la conexión SSH. Para hacerlo, escriba “exit” y salga de la shell remota:

exit

Debería estar de vuelta en el símbolo del sistema de su proyecto.

Utilice el directorio principal

Ahora, pruebe su directorio principal. El contenido de su directorio principal de Cloud Shell se encuentra en los proyectos de todas las sesiones de Cloud Shell, incluso después de que se finalice y reinicie la máquina virtual.

Cambie su directorio de trabajo actual:

cd $HOME

Use el editor de texto vi para abrir su archivo de configuración .bashrc.

vi ./.bashrc

El editor abrirá y mostrará el contenido del archivo. Presione la tecla ESC y, luego, escriba :wq para salir del editor.
