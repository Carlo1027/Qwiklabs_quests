Kubernetes Engine: Qwik Start

30 minutos 1 crédito
Calificar lab

GSP100

Descripción general

Google Kubernetes Engine (GKE) proporciona un entorno administrado para implementar, administrar y escalar sus aplicaciones en contenedores con la infraestructura de Google. El entorno de Kubernetes Engine consta de varias máquinas (específicamente, instancias de Google Compute Engine) que se agrupan para formar un clúster de contenedores. En este lab, adquirirá experiencia práctica en la creación de contenedores y la implementación de aplicaciones con GKE.

Organización de un clúster con Kubernetes Engine

Los clústeres de Kubernetes Engine funcionan con el sistema de administración de código abierto de Kubernetes. Este proporciona los mecanismos a través de los cuales interactúa con su clúster de contenedores. Los comandos y recursos de Kubernetes se usan para implementar y administrar aplicaciones, realizar tareas administrativas, establecer políticas y supervisar el estado de las cargas de trabajo implementadas.

Kubernetes se basa en los mismos principios de diseño que se utilizan para ejecutar algunos servicios populares de Google y ofrece los mismos beneficios: administración automática, supervisión y sondeos de funcionamiento de los contenedores de aplicaciones, ajuste de escala automático, actualizaciones progresivas y mucho más. Cuando ejecuta sus aplicaciones en un clúster de contenedores, está utilizando tecnología basada en la experiencia de más de 10 años que Google tiene ejecutando cargas de trabajo de producción en contenedores.

Kubernetes en Google Cloud Platform

Cuando ejecuta un clúster de Kubernetes Engine, también obtiene el beneficio de las funciones avanzadas de administración de clústeres que proporciona Google Cloud Platform, Las siguientes son algunas de ellas:

  • Balanceo de cargas para instancias de Compute Engine
  • Grupos de nodos para designar subconjuntos de nodos dentro de un clúster y así obtener flexibilidad adicional
  • Ajuste de escala automático del recuento de instancias de nodos del clúster
  • Actualizaciones automáticas para el software de nodo del clúster
  • Reparación automática de los nodos para mantenerlos disponibles y en buen estado
  • Registros y supervisión con Stackdriver para obtener una mayor visibilidad del clúster

Ahora que tiene un conocimiento básico de Kubernetes, aprenderá cómo implementar una aplicación en contenedores con Kubernetes Engine en menos de 30 minutos. Desplácese hacia abajo y siga los pasos que se muestran a continuación para configurar el entorno de su lab.

Configure una zona de procesamiento predeterminada

Una zona de procesamiento es una ubicación regional aproximada en la que residen sus clústeres y sus respectivos recursos. Por ejemplo, us-central1-a es una zona de la región us-central1.

Inicie una nueva sesión en Cloud Shell y ejecute el siguiente comando para configurar su zona de procesamiento predeterminada en us-central1-a:

gcloud config set compute/zone us-central1-a

Recibirá el siguiente resultado:

Updated property [compute/zone].

Cree un clúster de Kubernetes Engine

Un clúster consta de, al menos, una máquina de instancia principal del clúster y varias máquinas trabajadoras, llamadas nodos. Los nodos son instancias de máquinas virtuales (VM) de Compute Engine que ejecutan los procesos de Kubernetes necesarios para que sean parte del clúster.

Para crear un clúster, ejecute el siguiente comando. Para ello, reemplace [CLUSTER-NAME] por el nombre que eligió para el clúster (por ejemplo, my-cluster). Los nombres de los clústeres deben comenzar con una letra y terminar con un carácter alfanumérico, y no pueden tener más de 40 caracteres.

gcloud container clusters create [CLUSTER-NAME]

Puede omitir cualquier advertencia en el resultado. Podría llevar varios minutos terminar de crear el clúster. Algunos segundos después, debería obtener un resultado similar al siguiente:

NAME        LOCATION       ...   NODE_VERSION  NUM_NODES  STATUS
my-cluster  us-central1-a  ...   1.13.11-gke.9  3          RUNNING

Haga clic en Revisar mi progreso para verificar el objetivo. Crear un clúster de Kubernetes Engine

Obtenga las credenciales de autenticación para el clúster

Después de crear su clúster, debe obtener credenciales de autenticación para interactuar con él.

Para autenticar el clúster, ejecute el siguiente comando y reemplace [CLUSTER-NAME] por el nombre de su clúster:

gcloud container clusters get-credentials [CLUSTER-NAME]

Debería obtener un resultado similar al siguiente:

Fetching cluster endpoint and auth data.
kubeconfig entry generated for my-cluster.

Implemente una aplicación en el clúster

Ahora que ya creó el clúster, puede implementar una aplicación en contenedores en él. En este lab, ejecutará hello-app en su clúster.

Kubernetes Engine usa objetos de Kubernetes para crear y administrar los recursos de su clúster. Kubernetes proporciona el objeto de Implementación para implementar aplicaciones sin estado como servidores web. Los objetos de Servicio definen las reglas y el balanceo de cargas para acceder a su aplicación desde Internet.

Ejecute el siguiente comando kubectl create en Cloud Shell para crear un nuevo objeto de Implementación hello-server a partir de la imagen del contenedor hello-app:

kubectl create deployment hello-server --image=gcr.io/google-samples/hello-app:1.0

Debería recibir el siguiente resultado:

deployment.apps/hello-server created

Este comando de Kubernetes crea un objeto de Implementación que representa hello-server. En este caso, --image especifica la imagen de contenedor que se desea implementar. El comando obtiene la imagen de ejemplo desde un depósito de Google Container Registry. gcr.io/google-samples/hello-app:1.0 indica la versión específica de la imagen que se desea obtener. Si no se especifica una versión, se usará la más reciente.

Ahora, cree un Servicio de Kubernetes, que es un recurso de Kubernetes que le permite exponer su aplicación al tráfico externo. Para ello, ejecute el siguiente comando kubectl expose:

kubectl expose deployment hello-server --type=LoadBalancer --port 8080

En el ejemplo anterior, se ilustra lo siguiente:

  • --port especifica el puerto que expone el contenedor.
  • type="LoadBalancer" crea un balanceador de cargas de Compute Engine para su contenedor.

Debería recibir el siguiente resultado:

service/hello-server exposed

Haga clic en Revisar mi progreso para verificar el objetivo. Crear un nuevo objeto de Implementación - hello-server

Limpieza

Ejecute lo siguiente para borrar el clúster:

gcloud container clusters delete [CLUSTER-NAME]

Cuando se le indique, escriba Y para confirmar. Borrar el clúster puede llevar unos minutos. Para obtener más información sobre los clústeres borrados de Google Kubernetes Engine, consulte la documentación.
