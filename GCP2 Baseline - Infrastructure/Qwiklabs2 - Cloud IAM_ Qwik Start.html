Cloud IAM: Qwik Start

45 minutos 1 crédito
Calificar lab

GSP064

Descripción general

El servicio Identity and Access Management (IAM) de Google Cloud le permite crear y administrar permisos para los recursos de GCP. Cloud IAM unifica el control de acceso para los servicios de GCP en un solo sistema y proporciona un conjunto coherente de operaciones. En este lab práctico, aprenderá cómo asignar una función a un segundo usuario y cómo quitar las funciones asignadas que se asocian con Cloud IAM. Específicamente, accederá con 2 conjuntos de credenciales distintos para ver cómo se otorgan y revocan los permisos con las funciones de propietario y lector de proyecto de GCP.

Requisitos previos

Este es un lab de nivel introductorio. No es necesario que tenga conocimientos previos de Cloud IAM. Tener experiencia en Cloud Storage es útil para realizar las tareas de este lab, pero no es obligatorio contar con ella. Asegúrese de tener un archivo .txt o .html disponible. Si desea practicar habilidades más avanzadas de Cloud IAM, realice el siguiente lab:

  • Funciones personalizadas de IAM

Cuando esté listo, desplácese hacia abajo y siga los pasos para configurar su entorno de lab.

Configure dos usuarios

Como se mencionó antes, en este lab se proporcionan dos conjuntos de credenciales a fin de mostrar las políticas de IAM y los permisos disponibles para las funciones específicas.

En el panel ubicado en la parte izquierda de su lab, verá una lista de credenciales similares a las siguientes:

Observe que hay dos nombres de usuario: Username 1 y Username 2. Estos representan las identidades en Cloud IAM. Cada uno tiene distintos permisos de acceso asignados. Estas “funciones” de GCP establecen restricciones sobre lo que puede y no puede hacer con los recursos de GCP en el proyecto que se le asignó.

Acceda a GCP Console como el primer usuario

  1. Haga clic en el botón Abrir Google Console. Se abrirá una pestaña nueva del navegador. Si aparece el mensaje Selecciona una cuenta, haga clic en Usar otra cuenta.
  2. Se abrirá la página de acceso de GCP. Copie la credencial de Username 1, que se parece a googlexxxxxx_student@qwiklabs.net, y péguela en el campo "Correo electrónico o teléfono".
  3. Copie la contraseña del panel “Detalles de la conexión” y péguela en el campo de la contraseña de Acceso con Google.
  4. Haga clic en Siguiente y, luego, en Aceptar para aceptar las Condiciones del Servicio. Se abrirá Google Cloud Platform Console. Acepte las Condiciones del Servicio y marque No para no recibir actualizaciones por correo electrónico:

Acceda a GCP Console como el segundo usuario

  1. Vuelva a hacer clic en el botón Abrir Google Console. Se abrirá una pestaña nueva del navegador. Si aparece el mensaje Selecciona una cuenta, haga clic en Usar otra cuenta.
  2. Se abrirá la página de acceso de GCP. Copie la credencial de Username 2, que se parece a googlexxxxxx_student@qwiklabs.net, y péguela en el campo Correo electrónico o teléfono.
  3. Copie la contraseña desde el panel Detalles de la conexión y péguela en el campo de la contraseña de Acceso con Google.
  4. Haga clic en Siguiente y, luego, en Aceptar para aceptar las Condiciones del Servicio. Se abrirá Google Cloud Platform Console. Acepte las Condiciones del Servicio y marque No para no recibir actualizaciones por correo electrónico:

Ahora debería tener dos pestañas de GCP Console abiertas en su navegador: una en la que accedió como Username 1 y otra como Username 2.

Vea o restablezca el usuario en una pestaña del navegador

En algunos casos, es posible que se sobrescriba un usuario en una pestaña del navegador o que usted se confunda y no tenga claro qué usuario accedió en cada pestaña.

A fin de saber qué usuario accedió a una pestaña del navegador, desplácese sobre su avatar para ver su nombre de usuario en la pestaña.

Siga estos pasos para restablecer el usuario que accedió a una pestaña del navegador:

  1. Haga clic en su avatar y, luego, en Salir para salir.

  2. En el panel Detalles de la conexión, haga clic en Abrir Google Console y vuelva a acceder con el nombre de usuario y la contraseña adecuados.

La consola IAM y las funciones en el nivel del proyecto

  1. Vuelva a la página de GCP Console de Username 1.
  2. Seleccione el menú de navegación > IAM y administración > IAM. Ahora se encuentra en la consola “IAM y administración”.
  3. Haga clic en el botón +AGREGAR cerca de la parte superior de la página y, luego, en el menú desplegable “Selecciona una función” para explorar las funciones asociadas con los proyectos:

Debería ver las funciones de navegador, editor, propietario y lector. Estas cuatro funciones se conocen como funciones básicas en GCP. Establecen permisos a nivel del proyecto y, a menos que se especifique lo contrario, controlan el acceso a todos los servicios de GCP y su administración.

En la siguiente tabla, se extraen definiciones de la documentación de funciones de GCP que ofrecen una descripción general breve de los permisos de las funciones de navegador, lector, editor y propietario:

Nombre de la función

Permisos

roles/viewer

Permisos para acciones de solo lectura que no afectan el estado, como leer (pero no modificar) los recursos o datos existentes.

roles/editor

Todos los permisos de la función de lector y permisos adicionales para acciones que modifican el estado, como cambiar recursos existentes.

roles/owner

Todos los permisos de la función de editor, además de permisos para realizar las siguientes acciones:

  • Administrar funciones y permisos de un proyecto y todos los recursos dentro de este
  • Configurar la facturación de un proyecto

roles/browser (Beta)

Acceso de lectura para navegar en la jerarquía de un proyecto, incluida la carpeta, organización y política de Cloud IAM. Esta función no incluye el permiso para ver los recursos del proyecto.

Dado que pudimos administrar los permisos y funciones de este proyecto, Username 1 tiene permisos de propietario.

  1. Haga clic en CANCELAR para salir del panel “Agrega miembros”.

Prepare un recurso para probar el acceso

Asegúrese de estar en la consola de Username 1.

Cree un depósito

  1. Cree un depósito de GCS con un nombre único. En Console, seleccione el menú de navegación > Storage > Navegador.

  2. Haga clic en Crear depósito.

Nota: Si ve un mensaje de error de permisos durante la creación del depósito, salga y vuelva a acceder con las credenciales de Username 1.
  1. Actualice los siguientes campos y deje todos los demás con sus valores predeterminados:

Propiedad

Valor

Nombre:

nombre global único (créelo usted mismo)

Clase de almacenamiento predeterminada:

Multirregional

Tome nota del nombre del depósito. Lo usará en un paso posterior.

  1. Haga clic en Crear.
Nota: Si ve un mensaje de error de permisos durante la creación del depósito, salga y vuelva a acceder con las credenciales de Username 1.

Suba un archivo de muestra

  1. En la página Detalles del depósito, haga clic en el botón Cargar archivos.

  2. Busque en su computadora un archivo que pueda usar. Puede ser cualquier archivo de texto o html.

  3. Haga clic en los tres puntos al final de la línea que contiene el archivo y haga clic en Cambiar nombre.

  4. Cambie el nombre del archivo a "sample.txt".

  5. Haga clic en Cambiar nombre.

Haga clic en Revisar mi progreso para verificar el objetivo.

Cree un depósito y suba un archivo de muestra

Verifique el acceso del lector de proyecto

  1. Vaya a la consola de Username 2.

  2. Allí, seleccione el menú de navegación > Storage > Navegador. Verifique que el usuario pueda ver el depósito.

Username 2 tiene la función “Lector", con la que puede realizar acciones de solo lectura que no afectan el estado. En este ejemplo, se muestra la característica: el usuario puede ver los depósitos y archivos de Cloud Storage alojados en el proyecto de GCP a los que tiene acceso.

Quite el acceso al proyecto

Cambie a la consola de Username 1.

Quítele a Username 2 la función de lector de proyecto

  1. Seleccione el menú de navegación > IAM y administración > IAM. Luego, haga clic en el ícono de lápiz que se encuentra junto a Username 2.
Es posible que deba expandir la pantalla para poder ver el ícono de lápiz.

  1. Para quitar el acceso de lector de proyecto a Username 2, haga clic en el ícono de la papelera junto al nombre de la función. Luego, haga clic en GUARDAR.

Podrá ver que el usuario desapareció de la lista, por lo que ya no tiene acceso.

Nota: Los cambios pueden tardar hasta 80 segundos en propagarse y aplicarse. Obtenga más información aquí.

Verifique que Username 2 ya no tenga acceso

  1. Vaya a la consola de Username 2. Asegúrese de seguir usando las credenciales de Username 2 y de no haber salido del proyecto después de revocar los permisos. Si salió del proyecto, vuelva a acceder con las credenciales correctas.

  2. Seleccione el menú de navegación > Página principal.

  3. Seleccione el menú de navegación > Storage > Navegador para volver a Cloud Storage.

Debería ver un error de permisos como el siguiente:

Nota: Como se mencionó antes, es posible que la revocación de permisos tarde hasta 80 segundos. Si no vio un mensaje de error de permisos, espere 2 minutos y actualice la consola.

Haga clic en Revisar mi progreso para verificar el objetivo.

Quite el acceso al proyecto

Agregue permisos para Storage

  1. Copie el nombre de Username 2 desde el panel “Detalles de la conexión” de Qwiklabs.

  1. Vaya a la consola de Username 1. Asegúrese de seguir usando las credenciales de Username 1. Si salió del proyecto, vuelva a acceder con las credenciales correctas.

  2. En la consola, seleccione el menú de navegación > IAM y administración > IAM.

  3. Haga clic en +AGREGAR para pegar el nombre de Username 2 en el campo Nuevos miembros.

  4. En el campo Funciones, seleccione Storage > Lector de objetos de almacenamiento en el menú desplegable.

  5. Haga clic en GUARDAR.

Verifique el acceso

  1. Vaya a la consola de Username 2.

Username 2 no tiene la función de lector del proyecto, por lo que no puede ver el proyecto ni ninguno de sus recursos en Console. Sin embargo, este usuario tiene acceso específico a Cloud Storage. Veamos cómo funciona.

  1. Haga clic en el ícono de Activar Cloud Shell para abrir la línea de comandos de Cloud Shell.

  1. Abra una sesión de Cloud Shell y, luego, ingrese el siguiente comando para reemplazar [YOUR_BUCKET_NAME] con el nombre del depósito que creó:

gsutil ls gs://[YOUR_BUCKET_NAME]

  1. Debería obtener un resultado como el siguiente:

gs://[YOUR_BUCKET_NAME]/sample.txt
  1. Como puede ver, le otorgó acceso de lectura a Username 2 para el depósito de Cloud Storage.

Haga clic en Revisar mi progreso para verificar el objetivo.
