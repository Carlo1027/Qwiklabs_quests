Cloud Storage: Qwik Start - CLI/SDK

30 minutos 1 crédito
Calificar lab

GSP074

Resumen

Google Cloud Storage permite el almacenamiento y la recuperación de cualquier cantidad de datos en todo el mundo y en cualquier momento. Puede usar Google Cloud Storage en diversas situaciones, como entregar contenido de un sitio web, almacenar datos para archivarlos y recuperarlos en caso de desastres, o distribuir grandes objetos de datos a los usuarios a través de una descarga directa.

En este lab práctico, aprenderá a crear un depósito de almacenamiento, subir objetos a él, crear carpetas y subcarpetas en él y permitir el acceso público a los objetos con la línea de comandos de Google Cloud Platform.

Durante este lab, podrá verificar su trabajo en Console. Para ello, vaya al menú de navegación > Storage. Solo deberá actualizar su navegador después de ejecutar cada comando para ver los elementos nuevos que creó.

Cree un depósito

Ejecute el comando gsutil mb y reemplace "YOUR-BUCKET-NAME" por un nombre único para crear un depósito:

gsutil mb gs://YOUR-BUCKET-NAME/

Reglas de nomenclatura para los depósitos:

  • No incluya información confidencial en el nombre del depósito, ya que el espacio de nombres del depósito es global y tiene visibilidad pública.
  • Los nombres de depósitos solo pueden contener letras en minúscula, números, guiones (-), guiones bajos (_) y puntos (.). Los nombres que contienen puntos se deben verificar.
  • Deben comenzar y terminar con un número o una letra.
  • Pueden contener entre 3 y 63 caracteres. Los nombres que contienen puntos pueden tener hasta 222 caracteres, pero cada componente separado por un punto no puede tener más de 63 caracteres.
  • No se pueden representar como una dirección IP en notación decimal con puntos (por ejemplo, 192.168.5.4).
  • No pueden comenzar con el prefijo "goog".
  • No pueden contener "google" ni otras palabras similares a "google".
  • Además, para garantizar el cumplimiento de DNS y la compatibilidad en el futuro, no debería usar guiones bajos (_) ni un punto adyacente a otro punto o a un guion. Por ejemplo, "..", "-." o ".-" no son nombres válidos en DNS.

Si se ejecuta correctamente, el comando mostrará lo siguiente:

Creating gs://YOUR-BUCKET-NAME/...

Su depósito está listo y ya puede comenzar a almacenar su contenido.

Nota: Si usted o alguien más ya usa ese nombre para un depósito, el comando mostrará lo siguiente:

Creating gs://YOUR-BUCKET-NAME/... ServiceException: 409 Bucket YOUR-BUCKET-NAME already exists.

Vuelva a intentarlo con otro nombre.

Suba un objeto a su depósito

Ahora, suba un objeto al depósito.

En primer lugar, descargue esta imagen en una instancia temporal (ada.jpg) en Cloud Shell:

wget --output-document ada.jpg https://upload.wikimedia.org/wikipedia/commons/thumb/a/a4/Ada_Lovelace_portrait.jpg/800px-Ada_Lovelace_portrait.jpg

Use el comando gsutil cp para subir la imagen de la ubicación en la que la guardó al depósito que creó:

gsutil cp ada.jpg gs://YOUR-BUCKET-NAME
Sugerencia: Cuando ingrese el nombre del depósito, puede usar la tecla de tabulación para completarlo automáticamente.

Puede ver el proceso de carga de la imagen en la línea de comandos. Ya almacenó un objeto en su depósito.

Ahora puede quitar la imagen que descargó:

rm ada.jpg

Descargue un objeto de su depósito

Use el comando gsutil cp para descargar la imagen que almacenó en su depósito a Cloud Shell:

gsutil cp -r gs://YOUR-BUCKET-NAME/ada.jpg .

Si se ejecuta correctamente, el comando mostrará lo siguiente:

Copying gs://YOUR-BUCKET-NAME/ada.jpg...
/ [1 files][299.6 KiB/299.6 KiB]
Operation completed over 1 objects/299.6 KiB.

Ya descargó la imagen de su depósito.

Copie un objeto a una carpeta del depósito

Use el comando gsutil cp para crear una carpeta llamada image-folder y copie la imagen (ada.jpg) en ella:

gsutil cp gs://YOUR-BUCKET-NAME/ada.jpg gs://YOUR-BUCKET-NAME/image-folder/
Nota: Las carpetas de Google Cloud Storage tienen limitaciones en comparación con los sistemas de archivos locales, pero admiten muchas de las mismas operaciones.

Si se ejecuta correctamente, el comando mostrará lo siguiente:

Copying gs://YOUR-BUCKET-NAME/ada.jpg [Content-Type=image/png]...
- [1 files] [ 299.6 KiB/ 299.6 KiB]
Operation completed over 1 objects/299.6 KiB

El archivo de imagen ya se copió a una carpeta nueva de su depósito.

Pruebe la tarea completada

Haga clic en Revisar mi progreso para verificar la tarea realizada. Si subió correctamente un objeto a la carpeta mediante Cloud Storage, verá una puntuación de evaluación.

Copie un objeto a una carpeta del depósito (ada.jpg).

Muestre el contenido de un depósito o una carpeta

Use el comando gsutil ls para mostrar el contenido de un depósito:

gsutil ls gs://YOUR-BUCKET-NAME

Si se ejecuta correctamente, el comando mostrará un mensaje similar a este:

gs://YOUR-BUCKET-NAME/ada.jpg
gs://YOUR-BUCKET-NAME/image-folder/

Ese es todo el contenido que se encuentra en su depósito actualmente.

Muestre los detalles de un objeto

Use el comando gsutil ls con la marca -l para obtener algunos detalles del archivo de imagen que subió a su depósito:

gsutil ls -l gs://YOUR-BUCKET-NAME/ada.jpg

Si se ejecuta correctamente, el comando mostrará un mensaje similar a este:

306768  2017-12-26T16:07:570Z  gs://YOUR-BUCKET-NAME/ada.jpg
TOTAL: 1 objects, 30678 bytes (299.58 KiB)

Ahora conoce el tamaño y la fecha de creación de la imagen.

Permita el acceso público a su objeto

Use el comando gsutil acl ch para que todos los usuarios tengan permisos de lectura para el objeto almacenado en su depósito:

gsutil acl ch -u AllUsers:R gs://YOUR-BUCKET-NAME/ada.jpg

Si se ejecuta correctamente, el comando mostrará lo siguiente:

Updated ACL on gs://YOUR-BUCKET-NAME/ada.jpg

Su imagen ahora es pública y se puede poner a disposición de todo el mundo.

Pruebe la tarea completada

Haga clic en Revisar mi progreso para verificar la tarea realizada. Si compartió correctamente un objeto desde su depósito de almacenamiento, verá una puntuación de evaluación.

Quite el acceso público

Para quitar este permiso, ejecute el siguiente comando:

gsutil acl ch -d AllUsers gs://YOUR-BUCKET-NAME/ada.jpg

Si se ejecuta correctamente, el comando mostrará lo siguiente:

Updated ACL on gs://YOUR-BUCKET-NAME/ada.jpg

Ya quitó el acceso público a este objeto. Para comprobarlo, puede hacer clic en el botón Actualizar de Console. Se quitará la marca de verificación. Si vuelve a cargar la página con la imagen, verá un mensaje de error.

Borre objetos

Use el comando gsutil rm para borrar un objeto; en este caso, el archivo de imagen de su depósito:

gsutil rm gs://YOUR-BUCKET-NAME/ada.jpg

Si se ejecuta correctamente, el comando mostrará lo siguiente:

Removing gs://YOUR-BUCKET-NAME/ada.jpg...

Actualice Console. La copia del archivo de imagen ya no está almacenada en Cloud Storage (sin embargo, todavía existe la copia que hizo en la carpeta image-folder/).
