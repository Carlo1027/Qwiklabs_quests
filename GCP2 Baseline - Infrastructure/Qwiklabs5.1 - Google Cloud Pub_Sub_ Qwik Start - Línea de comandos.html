Google Cloud Pub/Sub: Qwik Start - Línea de comandos

40 minutos Gratis
Calificar lab

GSP095

Resumen

Google Cloud Pub/Sub es un servicio de mensajería para intercambiar datos de eventos entre aplicaciones y servicios. La separación de emisores y receptores permite una comunicación segura y de alta disponibilidad entre las aplicaciones escritas de forma independiente. Google Cloud Pub/Sub ofrece una mensajería duradera y de baja latencia, que los desarrolladores suelen usar para implementar flujos de trabajo asíncronos, distribuir notificaciones de eventos y transmitir datos desde varios procesos o dispositivos.

Actividades

En este lab, hará lo siguiente:

  • Aprenderá los conceptos básicos de Pub/Sub.

  • Creará, borrará y mostrará una lista de temas de Pub/Sub.

  • Creará, borrará y mostrará una lista de suscripciones de Pub/Sub.

  • Publicará mensajes en un tema.

  • Usará un suscriptor de extracción para mostrar mensajes individuales de temas.

  • Usará un suscriptor de extracción con una marca para mostrar varios mensajes.

Requisitos previos

Este es un lab de nivel introductorio. Eso quiere decir que se da por hecho que tiene poca o ninguna experiencia con Pub/Sub. Se le enseñarán los conceptos básicos de la configuración y el uso de este servicio de GCP.

Antes de realizar este lab, tenga en cuenta su nivel de competencia en Pub/Sub. A continuación, le mostramos una lista de labs más avanzados que le permitirán aplicar sus conocimientos de Pub/Sub a diferentes casos prácticos y servicios de nube:

Pub/Sub: Conceptos básicos

Como se mencionó anteriormente, Google Cloud Pub/Sub es un servicio de mensajería global asíncrono. Hay tres términos que aparecen a menudo en Pub/Sub: temas, publicar y suscribirse.

Un tema es una string compartida que permite a las aplicaciones conectarse entre sí mediante un subproceso común.

Los publicadores envían (o publican) un mensaje a un tema de Cloud Pub/Sub. Luego, los subscriptores se "suscribirán" a ese subproceso, desde el que podrán extraer mensajes del tema o configurar webhooks para suscripciones push. Cada suscriptor debe confirmar cada mensaje dentro de un período configurable.

En resumen, un publicador crea y envía mensajes a un tema, y un suscriptor crea una suscripción a un tema para recibir mensajes de él.

Temas de Pub/Sub

Pub/Sub viene preinstalado en Google Cloud Shell, por lo que no es necesario realizar instalaciones o configuraciones para comenzar a usar este servicio.

Lo primero que debemos hacer es crear un tema. Ejecute el siguiente comando para crear un tema que se llame myTopic:

gcloud pubsub topics create myTopic

Pruebe la tarea completada

Haga clic en Revisar mi progreso para verificar la tarea realizada. Si la completó correctamente, se le otorgará un puntaje de evaluación.

Cree un tema de Pub/Sub.

Por si acaso, creemos otros dos temas, uno llamado Test1 y otro llamado Test2:

gcloud pubsub topics create Test1

gcloud pubsub topics create Test2

Ahora, veamos los tres temas que acabamos de crear. Ejecute el siguiente comando:

gcloud pubsub topics list

El resultado debería ser similar al siguiente:

name: projects/qwiklabs-gcp-3450558d2b043890/topics/myTopic
---
name: projects/qwiklabs-gcp-3450558d2b043890/topics/Test2
---
name: projects/qwiklabs-gcp-3450558d2b043890/topics/Test1

Es hora de limpiar. Borremos Test1 y Test2 mediante los siguientes comandos:

gcloud pubsub topics delete Test1

gcloud pubsub topics delete Test2

Ejecute el comando gcloud pubsub topics list una vez más. Debería obtener el siguiente resultado:

---
name: projects/qwiklabs-gcp-3450558d2b043890/topics/myTopic

Suscripciones Pub/Sub

Ahora que ya sabemos cómo crear, ver y borrar temas, trabajemos con las suscripciones.

Ejecute el siguiente comando para crear una suscripción que se llame mySubscription en el tema myTopic:

gcloud  pubsub subscriptions create --topic myTopic mySubscription

Pruebe la tarea completada

Haga clic en Revisar mi progreso para verificar la tarea realizada. Si la completó correctamente, se le otorgará un puntaje de evaluación.

Cree una suscripción de Pub/Sub.

Agreguemos otras dos suscripciones a myTopic. Ejecute los siguientes comandos para crear las suscripciones Test1 y Test2:

gcloud  pubsub subscriptions create --topic myTopic Test1

gcloud  pubsub subscriptions create --topic myTopic Test2

Asegurémonos de que se crearon nuestras suscripciones. Ejecute el siguiente comando para mostrar las suscripciones a myTopic:

gcloud pubsub topics list-subscriptions myTopic

El resultado debería ser similar al siguiente:

---
  projects/qwiklabs-gcp-3450558d2b043890/subscriptions/Test2
---
  projects/qwiklabs-gcp-3450558d2b043890/subscriptions/Test1
---
  projects/qwiklabs-gcp-3450558d2b043890/subscriptions/mySubscription

Ponga a prueba sus conocimientos

Ahora, borremos las suscripciones Test1 y Test2. Ejecute los siguientes comandos:

gcloud pubsub subscriptions delete Test1

gcloud pubsub subscriptions delete Test2

Verifiquemos si se borraron las suscripciones Test1 y Test2. Ejecute el comando gcloud pubsub topics list-subscriptions myTopic una vez más. Debería obtener el siguiente resultado:

Cómo publicar y extraer un solo mensaje con Pub/Sub

Ahora que ya sabemos cómo crear, ver y borrar suscripciones, veamos cómo publicar un mensaje en un tema de Pub/Sub.

Ejecute el siguiente comando para publicar el mensaje "hello" en el tema que creamos anteriormente (myTopic):

gcloud pubsub topics publish myTopic --message "Hello"

Publiquemos algunos mensajes más en myTopic. Para ello, ejecute los siguientes comandos (reemplace <YOUR NAME> con su nombre y <FOOD> con una comida que le guste):

gcloud pubsub topics publish myTopic --message "Publisher's name is <YOUR NAME>"

gcloud pubsub topics publish myTopic --message "Publisher likes to eat <FOOD>"

gcloud pubsub topics publish myTopic --message "Publisher thinks Pub/Sub is awesome"

Ahora, usaremos el comando pull para obtener los mensajes desde nuestro tema. El comando pull está basado en la suscripción; es decir que debería funcionar porque configuramos una suscripción llamada mySubscription en el tema myTopic previamente.

Use el siguiente comando para extraer los mensajes que acabamos de publicar desde el tema de Pub/Sub:

gcloud pubsub subscriptions pull mySubscription --auto-ack

El resultado debería ser similar al siguiente:

¿Qué sucede aquí? Publicamos 4 mensajes en nuestro tema, pero solo se mostró 1 en la salida.

En este punto es importante tener en cuenta un par de funciones del comando pull que a menudo confunden a los desarrolladores:

  • Si se usa el comando pull sin marcas, solo se mostrará un mensaje en la salida, incluso si está suscrito a un tema que tiene más mensajes.
  • Una vez que se muestra un mensaje individual en la salida desde un comando pull basado en la suscripción, no podrá acceder a ese mensaje nuevamente con ese comando.

Para comprender mejor a qué se refiere el segundo punto, pruebe ejecutar el último comando tres veces más. Verá que se mostrarán los mensajes que publicó antes. Ahora, ejecútelo una cuarta vez y verá el siguiente resultado (ya que no había más para mostrar):

gcpstaging20394_student@cloudshell:~ (qwiklabs-gcp-3450558d2b043890)$ gcloud pubsub subscriptions pull mySubscription --auto-ack
Listed 0 items.

En la siguiente y última sección, aprenderá cómo extraer varios mensajes desde un tema con una marca o flag.

Cómo extraer todos los mensajes desde las suscripciones con Pub/Sub

Puesto que ya extrajimos todos los mensajes desde nuestro tema en el último ejemplo, ahora publiquemos algunos mensajes más en myTopic. Ejecute los siguientes comandos:

gcloud pubsub topics publish myTopic --message "Publisher is starting to get the hang of Pub/Sub"

gcloud pubsub topics publish myTopic --message "Publisher wonders if all messages will be pulled"

gcloud pubsub topics publish myTopic --message "Publisher will have to test to find out"

Ahora que ya se publicaron mensajes en nuestro tema, agreguemos una marca o flag a nuestro comando para mostrar los tres mensajes mediante la ejecución de una sola línea de comandos. Es posible que no lo haya notado, pero estuvimos usando una marca en los ejemplos anteriores: --auto-ack, que es parte del comando pull. Es una marca que muestra nuestros mensajes en el formato de cuadro que vemos en los mensajes extraídos.

limit es otra marca que configura un límite superior para la cantidad de mensajes que queremos extraer. Ejecute el comando pull con la marca limit:

gcloud pubsub subscriptions pull mySubscription --auto-ack --limit=3

El resultado debería ser similar al siguiente:

Excelente. Aprendimos cómo agregar marcas a un comando de Pub/Sub para mostrar un grupo más grande de mensajes. Está en buen camino para convertirse en un experto en Pub/Sub.
