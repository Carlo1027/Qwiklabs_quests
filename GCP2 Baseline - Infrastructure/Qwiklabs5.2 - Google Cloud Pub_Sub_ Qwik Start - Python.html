Google Cloud Pub/Sub: Qwik Start - Python

30 minutos 1 crédito
Calificar lab

Descripción general

El servicio Google Cloud Pub/Sub permite que las aplicaciones intercambien mensajes de forma confiable, rápida y asíncrona. Para ello, un productor de datos publica mensajes en un tema de Cloud Pub/Sub. Luego, un cliente suscriptor crea una suscripción al tema y consume mensajes desde la suscripción. Cloud Pub/Sub conserva por un máximo de siete días los mensajes que no se pudieron entregar de forma confiable.

En este lab, se muestra cómo comenzar a publicar mensajes con Cloud Pub/Sub mediante la biblioteca cliente Python.

Actividades

En este lab, realizará lo siguiente:

  • Aprenderá los conceptos básicos de Pub/Sub.

  • Creará y mostrará un tema de Pub/Sub.

  • Creará y mostrará una suscripción de Pub/Sub.

  • Publicará mensajes en un tema.

  • Usará un suscriptor de extracción para obtener mensajes de temas específicos.

Conceptos básicos de Pub/Sub

Google Cloud Pub/Sub es un servicio de mensajería global asíncrono. Existen tres términos que aparecen con frecuencia en Pub/Sub: temas, publicar y suscribirse.

Un tema es una string compartida que permite que las aplicaciones se conecten entre sí a través de una conversación en común.

Los publicadores envían (o publican) mensajes en un tema de Cloud Pub/Sub. Luego, los suscriptores se “suscriben” a esa conversación, en la que pueden extraer mensajes del tema o configurar webhooks para suscripciones push. Cada suscriptor debe confirmar cada mensaje dentro de un período configurable.

En resumen, un publicador crea y envía mensajes a un tema, y un suscriptor crea una suscripción a un tema para recibir mensajes de él.

Pub/Sub en GCP

Pub/Sub viene preinstalado en Google Cloud Shell, por lo que no es necesario realizar instalaciones o configuraciones para comenzar a usar este servicio. En este lab, usará Python a fin de crear el tema y el suscriptor y, luego, para ver el mensaje. Utilizará un comando de gcloud para publicar el mensaje en el tema.

Cree un tema

A fin de publicar datos en Cloud Pub/Sub, debe crear un tema y, luego, configurar un publicador para él.

Configure la variable de entorno GOOGLE_CLOUD_PROJECT. Puede encontrar el ID de su proyecto de GCP en la página CONNECTION DETAILS.

export GLOBAL_CLOUD_PROJECT=GCP Project ID

publisher.py es una secuencia de comandos que demuestra cómo realizar operaciones básicas en temas con la API de Cloud Pub/Sub. Vea el contenido de la secuencia de comandos del publicador:

cat publisher.py

De forma alternativa, puede usar los editores de shell que están instalados en Cloud Shell, como nano o vim, o el editor de código de Cloud Shell para ver python-docs-samples/pubsub/cloud-client/publisher.py.

Publique mensajes

Ahora que configuró MyTopic (el tema) y una suscripción a MyTopic (MySub), intente usar los comandos de gcloud para publicar un mensaje en MyTopic.

Publique el mensaje “Hello” en MyTopic.

gcloud pubsub topics publish MyTopic --message "Hello"

Publique otros mensajes en MyTopic. Para ello, ejecute los siguientes comandos (reemplace <YOUR NAME> con su nombre y <FOOD> con una comida que le guste):

gcloud pubsub topics publish MyTopic --message "Publisher's name is <YOUR NAME>"

gcloud pubsub topics publish MyTopic --message "Publisher likes to eat <FOOD>"

gcloud pubsub topics publish MyTopic --message "Publisher thinks Pub/Sub is awesome"

Vea los mensajes

Ahora que publicó mensajes en MyTopic, extraiga y vea los mensajes con MySub.

Use MySub para extraer el mensaje de MyTopic:

python subscriber.py $GLOBAL_CLOUD_PROJECT receive MySub
