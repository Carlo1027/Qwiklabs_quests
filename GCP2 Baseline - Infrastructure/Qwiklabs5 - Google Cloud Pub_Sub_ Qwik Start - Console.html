Google Cloud Pub/Sub: Qwik Start - Console

30 minutos Gratis
Calificar lab

GSP096

Descripción general

Google Cloud Pub/Sub es un servicio de mensajería para intercambiar datos de eventos entre aplicaciones y servicios. Un productor de datos publica mensajes en un tema de Cloud Pub/Sub. Un consumidor crea una suscripción a ese tema. Los suscriptores pueden extraer mensajes desde una suscripción o se configuran como webhooks para suscripciones push. Cada suscriptor debe confirmar cada mensaje dentro de un período configurable.

Actividades

  • Configurar un tema para conservar datos
  • Suscribirse a un tema para acceder a los datos
  • Publicar y consumir mensajes con un suscriptor de extracción

Configuración y requisitos

Configuración de Qwiklabs

Antes de hacer clic en el botón Comenzar lab

Lea estas instrucciones. Los labs tienen un tiempo límite; no puede ponerlos en pausa. El cronómetro, que comienza a funcionar cuando hace clic en Comenzar lab, indica por cuánto tiempo estarán los recursos en la nube a su disposición.

Este lab práctico de Qwiklabs le permitirá llevar a cabo las actividades correspondientes en un entorno de nube real, no en uno de simulación o demostración. Para ello, recibe nuevas credenciales temporales que utilizará para acceder a Google Cloud Platform durante el transcurso del lab.

Lo que necesita

Para completar este lab, necesitará lo siguiente:

  • Acceso a un navegador de Internet estándar. Se recomienda el navegador Chrome
  • Tiempo para completar el lab

Nota: Si ya tiene un proyecto o una cuenta personal de GCP, no los use para este lab.

Ahora que comenzó su lab, accederá a la consola de Google Cloud Shell y, luego, iniciará la herramienta de línea de comandos.

Cómo iniciar su lab y acceder a Console

  1. Haga clic en el botón Comenzar lab. Si debe pagar por el lab, se abrirá una ventana emergente para que seleccione su forma de pago. En la parte izquierda, verá un panel con las credenciales temporales que debe usar para este lab.

  2. Copie el nombre de usuario y, luego, haga clic en Abrir Google Console. El lab inicia los recursos y abre otra pestaña que muestra la página Seleccione una cuenta.

    Sugerencia: Abra las pestañas en ventanas separadas, una junto a la otra.

  3. En la página Seleccione una cuenta, haga clic en Usar otra cuenta.

  4. Se abrirá la página de acceso. Pegue el nombre de usuario que copió del panel Detalles de conexión. Luego, copie y pegue la contraseña.

    Importante: Debe usar las credenciales del panel Detalles de conexión. No use sus credenciales de Qwiklabs. Si tiene una cuenta propia de GCP, no la use para este lab (evite incurrir en cargos).

  5. Avance por las siguientes páginas haciendo clic en ellas:

    • Acepte los Términos y Condiciones.
    • No agregue opciones de recuperación o autenticación de dos factores (esta es una cuenta temporal).
    • No se registre para obtener pruebas gratuitas.

Después de un momento, se abrirá GCP Console en esta pestaña.

Nota: Para ver el menú con una lista de productos y servicios de GCP, haga clic en Menú de navegación, en la esquina superior izquierda, junto a "Google Cloud Platform".

  1. En GCP Console, en la barra de herramientas superior derecha, haga clic en el botón Abrir Cloud Shell.

  2. Haga clic en Continue (Continuar):

Configure Pub/Sub

Puede usar la consola de Google Cloud Shell para ejecutar operaciones en Google Cloud Pub/Sub.

A fin de usar Pub/Sub, debe crear un tema que conserve datos y una suscripción para acceder a los datos publicados en el tema.

  1. Haga clic en el Menú de navegación > Pub/Sub > Temas.

  1. Haga clic en Crear Tema.

  1. El tema deberá tener un nombre único. Para este laboratorio, nombre su tema MyTopic. En el Crear un tema dialog:
  • Nombre su tema MyTopic.
  • Deje Encriptado con el valor predeterminado.
  • Haga clic en Crear un tema

Acaba de crear un tema.

Pruebe la tarea completada

Haga clic en Revisar mi progreso para verificar la tarea realizada. Si creó correctamente el tema de Cloud Pub/Sub, verá una puntuación de evaluación.

Crea un tema de Pub/Sub.

Agregue una suscripción

Ahora, creará una suscripción para acceder al tema.

  1. Haga clic en Temas en el panel izquierdo para devolverse a el dialogo Temas. Para el tema deberá hacer clic en el icono de los tres puntos > Crear suscripción.

  1. En el diálogo Añadir una suscripción al tema:
  • Escriba un nombre para la suscripción, como MySub.
  • Configure el tipo de entrega a Extracción.
  • Deje las otras opciones con los valores predeterminados.
  • Haga clic en Create.

Sus suscripciones están listadas en la lista de Suscripciones.

Pruebe la tarea completada

Haga clic en Revisar mi progreso para verificar la tarea realizada. Si creó correctamente una suscripción para el tema de Cloud Pub/Sub, verá una puntuación de evaluación.

Agregue una suscripción.

Ponga a prueba sus conocimientos

A continuación, se presentan algunas preguntas de opción múltiple para reforzar sus conocimientos de los conceptos de este lab. Trate de responderlas.

Publique un mensaje en el tema

  1. En la parte superior del diálogo Detalles del tema, haga clic en Publicar Mensaje. Es posible que tenga que ampliar la ventana del navegador para ver la opción Publicar Mensaje.

  1. Ingrese Hello World en el campo Mensaje y haga clic en Publicar.

Vea el mensaje

A fin de ver el mensaje, usará la suscripción (MySub) para extraer el mensaje (Hello World) desde el tema (MyTopic).

Ingrese el siguiente comando en la línea de comandos.

gcloud pubsub subscriptions pull --auto-ack MySub

El mensaje aparece en el campo DATA del resultado del comando.

Creó un tema de Pub/Sub, publicó en el tema, creó una suscripción y, luego, usó la suscripción para extraer datos del tema.
